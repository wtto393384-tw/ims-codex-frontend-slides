# PBL 簡報素材：IMS 進銷存系統開發流程

## 專題定位

本專題以中小型公司內部使用的簡易進銷存系統為題，採用 Flask + SQLite + Bootstrap 建立可展示、可操作、可匯出報表的 Web 系統。開發目標不是大型 ERP，而是完成 PBL 展示所需的核心流程：商品、分類、廠商、客戶、進貨、銷貨、庫存、庫存異動與 Excel 報表。

## 技術選型

- 後端：Python Flask
- 資料庫：SQLite
- 前端：Bootstrap + 自訂 CSS，參照 Airtable 風格的清楚表格與側邊導覽
- 報表：openpyxl 產生 Excel 檔案
- 測試：Python smoke tests + Chrome 實機頁面測試
- AI 協作：需求拆解、資料表設計、功能實作、錯誤修正、測試腳本、簡報素材整理

## 系統核心功能

- 帳號登入與帳號管理
- 產品分類管理
- 商品管理 CRUD
- 廠商管理 CRUD
- 客戶管理 CRUD
- 進貨管理：一張進貨單可包含多筆商品
- 銷貨管理：一張銷貨單可包含多筆商品，並檢查庫存不足
- 庫存查詢：依商品、分類、庫存狀態查詢
- 庫存異動紀錄：追蹤進貨入庫、銷貨出庫
- Excel 報表匯出：個別報表與 IMS_總報表.xlsx
- Demo seed data：展示用資料，可重複建立且避免重複寫入

## 已完成驗證重點

- Flask 服務可正常啟動，健康檢查 `/health` 正常。
- Chrome plugin 可開啟並測試主要頁面。
- 主要功能頁無 Traceback、Internal Server Error 或 SQLite 錯誤。
- Excel 總報表可下載，檔案名稱為 `IMS_總報表.xlsx`。
- Excel 總報表包含 5 個工作表：商品庫存、進貨明細、銷貨明細、庫存異動、低庫存。
- 使用者已人工確認 Excel 檔案內資料正常。

## 展示用說明詞

本系統採用 Web 模式，使用者只需要透過瀏覽器操作，不需要在每台電腦另外安裝單機版程式。SQLite 不需要額外啟動資料庫服務，資料會儲存在專案內的資料庫檔案中，適合 PBL 專題展示與小型內部系統原型。

AI 在本專題中主要扮演共同開發者角色，協助將需求拆成可實作功能，設計資料表，產生 Flask 路由、HTML 表單、查詢條件、Excel 報表，並用測試腳本與瀏覽器實測驗證功能是否正確。
