FlaskSQLiteBootstrap UIExcel 報表AI 協作
Problem Context

沒有大型 ERP，也要能管好商品、交易與庫存。

中小型企業常見痛點是資料分散、庫存難追、報表人工整理。IMS 專題用 Web 模式解決展示需求：使用者只要透過瀏覽器操作，不需在每台電腦安裝單機程式；SQLite 不需要額外啟動資料庫服務。

01
商品
分類、價格、單位、安全庫存集中管理。
02
進貨
一張進貨單包含多筆商品，建立後自動入庫。
03
銷貨
建立銷貨單前檢查庫存，避免超賣。
04
報表
匯出 Excel，讓成果能被檢查與展示。

Debug Loop

!錯誤回貼範例

執行銷貨單建立時出現 Internal Server Error，Traceback 指向 `deduct_stock()`。請分析是否是庫存欄位型別、交易順序或表單欄位名稱不一致，並提供修正版。

Workflow

功能流程：主檔建立後，交易才有資料可用。

進貨入庫
登入系統 → 建立商品、分類、廠商 → 建立進貨單 → 選擇多筆商品 → 增加庫存 → 寫入庫存異動。
銷貨出庫
建立客戶 → 建立銷貨單 → 檢查庫存是否足夠 → 足夠則扣庫存，不足則阻擋建立並提示。
庫存查詢
依商品、分類、狀態查詢目前庫存，並標示低庫存商品。
報表輸出
依資料表查詢結果，由 openpyxl 產生 Excel 總報表與個別報表。
Inventory Logic

進貨、銷貨與庫存的關聯

進貨入庫
1. 建立商品、分類與廠商主檔
2. 建立一張多品項進貨單
3. 系統增加商品目前庫存
4. 寫入庫存異動：進貨入庫
銷貨出庫
1. 建立客戶並開立銷貨單
2. 系統先檢查庫存是否足夠
3. 不足時阻擋，足夠時扣庫存
4. 庫存查詢與低庫存報表同步更新

Platform Comparison

自建 Flask IMS 與 TurboPlus 平台對比

面向自建 Flask 進銷存系統威聯網 TurboPlus 平台
建置方式從需求、資料表、路由、畫面到報表自行開發。可視為平台化或 Low-code / No-code 建置工具，速度通常較快。
優點架構輕量、Web 模式、SQLite 可隨專案攜帶、Excel 報表可客製。可能具備企業級穩定度、既有模組與平台維護能力。
限制需要 Python、Flask、HTML 與除錯能力。可能有授權費用，客製化與底層邏輯修改受平台限制。
教學價值能理解系統架構、資料庫關聯、商業邏輯與測試驗證。較適合理解平台導入與快速建置。
Why Build From Scratch

PBL 選擇從零開始，是為了學會系統怎麼被做出來。

平台工具能快速產出結果；自建系統則讓學生看見需求如何轉成資料表、資料表如何支撐功能、功能如何被測試，這才是專題導向學習最有價值的地方。
